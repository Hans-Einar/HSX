# Project Design Overview — HSX Runtime & Tooling

> Overview of detailed design artifacts. Each sub-document owns the authoritative specification for its domain.

## Design Catalogue
- **MiniVM (`(4.1)vm.md`):** Interpreter architecture, context model, ABI rules, and implementation guidance for the MiniVM core.
- **Executive (`(4.2)executive.md`):** Scheduler semantics, session/event streaming, provisioning hooks, and executive-side services.
- **Mailbox subsystem (`(4.3)mailbox.md`):** Message semantics, descriptor management, wait/wake flow, and resource quotas.
- **Value & Command layer (`(4.4)val_cmd.md`):** Value registration, command dispatch, persistence policies, and transport bindings.
- **Toolchain & artefacts (`(4.5)toolchain.md`):** Assembler/linker behaviour, .hxe artefacts, metadata outputs, and build pipeline expectations.
- **Toolkit (`(4.6)toolkit.md`):** CLI/TUI debugger design, event handling, watch/breakpoint workflow, and packaging strategy.
- **Provisioning & Persistence (`(4.7)provisioning.md`):** CAN/SD flows, FRAM layout, provisioning lifecycle, and recovery paths.

## Guidelines
- Each design doc must define **Preconditions** (system state/services required) and **Postconditions** (capabilities delivered) for implementation planning.
- Reference study design goals (DG) and options (DO) once `main/(2)study.md` tagging lands (see `main/studyReferenceRework.md`).
- Keep cross-document updates coordinated: when changing protocol/resource assumptions, update the relevant design doc, architecture view, and refs (e.g., `docs/executive_protocol.md`, `docs/resource_budgets.md`).
- Track progress via `main/(4)design/playbook.md` and capture implementation follow-ups in `main/refactorNotes.md`.
